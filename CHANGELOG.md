# Полный технический ченджлог

## ⏱ Динамические тайминги

### Проблема
Раньше шаги капчи выполнялись с фиксированными задержками. VK детектил паттерн и возвращал `status: "BOT"`.

### Решение
Введена система `CaptchaSessionTiming` — **единая сессия таймингов** на всё решение капчи. Каждый шаг получает случайную задержку из реалистичного диапазона:

- **Загрузка и чтение** — человек не кликает мгновенно, сначала осматривает страницу
- **Движение к виджету** — пауза между изучением и первым действием
- **Путь к чекбоксу** — самый важный промежуток: движение мыши + пауза перед кликом
- **Ожидание результата** — проверка VK не мгновенна, человек ждёт
- **«Зависания»** — 10% шанс дополнительной паузы, потому что люди иногда «тупят»

**Гарантии:**
- ✅ Суммарное время — в диапазоне 5–10 секунд (не слишком быстро, не слишком медленно)
- ✅ Каждый шаг — **динамическая вариация**, никогда не повторяется одинаково
- ✅ Самый длинный промежуток — между componentDone и check (как у реального человека)

**Результат:** Статус `BOT` перестал появляться. Все последние попытки прошли успешно.

## 🖱 Реалистичный курсор

### Было
Курсор генерировался без привязки к реальному поведению человека.

### Стало — `GenerateCaptchaCursor()`
- **Кубическая кривая Безье** — плавная дуговая траектория к чекбоксу «Я не робот»
- **Микро-дрожание руки** — реальный человек не ведёт мышь идеально ровно
- **Случайная начальная позиция** — человек двигает мышь из разных точек экрана
- **Разное количество точек** — иногда больше, иногда меньше, как в жизни

## 📱 Мобильные сенсоры (акселерометр/гироскоп)

### Было
- Сенсоры отправлялись не всегда
- Данные были полностью рандомными — без физики

### Стало
- ✅ Отправляются **всегда** (100% запросов)
- ✅ **Базовая ориентация** — телефон наклонён к лицу, гравитация распределяется по осям Y и Z
- ✅ **Инерция** — значения плавно дрейфуют, а не скачут хаотично
- ✅ **Микро-тремор рук** — коррелированный шум по всем осям (палец дрожит)
- ✅ **Гироскоп с памятью** — предыдущее значение влияет на следующее

## 👆 Тапы (прикосновения к экрану)

### Было
- Простые `{x, y, duration}` без временных меток
- Фиксированное количество

### Стало
- ✅ Добавлено поле **`time`** — миллисекунды от начала сессии (реальный Android отправляет timestamp)
- ✅ **Разные сценарии:** от 0 тапов (человек просто смотрит) до 6+ (активное пользование)
- ✅ **Первый тап не мгновенный** — человек сначала осматривается
- ✅ **Интервалы между тапами** варьируются — не робот

## 📊 Статичные фингерпринты

### Было
- `debug_info` генерировался **случайным** каждый запрос — это красный флаг для VK
- Не было Canvas, WebGL, AudioContext

### Стало
| Фингерпринт | Поведение |
|-------------|-----------|
| `debug_info` | **СТАТИЧНЫЙ** — привязан к DeviceID через SHA-256 |
| `canvasFP` | **СТАТИЧНЫЙ** — зависит от GPU устройства |
| `webglFP` | **СТАТИЧНЫЙ** — GPU name + хэш |
| `audioFP` | **СТАТИЧНЫЙ** — AudioContext fingerprint |

## 🔐 HTTP-заголовки для десктопной капчи

Добавлен полный набор заголовков, которые реальный Chrome отправляет при запросе к VK API:
- `sec-ch-ua-*` (платформа, версия браузера, мобильность)
- `Sec-Fetch-*` (site, mode, dest)
- `DNT`, `Priority`, `Accept-Language`

## 🔄 Поток выполнения капчи

```
1. Загрузка HTML страницы капчи
2. Человек «осматривает» страницу (пауза)
3. Решение PoW (мгновенно)
4. Пауза перед действиями API
5. captchaNotRobot.settings
6. Пауза — изучение виджета
7. captchaNotRobot.componentDone
8. Пауза — движение мыши + клик к чекбоксу ★
9. captchaNotRobot.check (главный запрос)
10. Пауза — ожидание результата
11. captchaNotRobot.endSession

★ Самый важный шаг — VK анализирует этот промежуток особенно внимательно
```
