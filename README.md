## 🧠 Идея

> Операторы связи и ТСПУ активно блокируют VPN-трафик. Но звонки в VK — это **святое**.

WDTT прокладывает WireGuard-туннель **внутри** DTLS-соединения через TURN-серверы VK. Для оператора ваш трафик неотличим от обычного видеозвонка — тот же протокол, те же серверы, те же порты.

<br>

## 🏗 Архитектура

### Схема подключения

```
╔══════════════════╗          DTLS / UDP           ╔══════════════════╗         UDP          ╔══════════════════╗
║                  ║     (имитация звонка VK)      ║                  ║     (WireGuard)      ║                  ║
║   📱 Клиент      ║ ═══════════════════════════► ║  ☁️  VK TURN      ║ ═══════════════════► ║  🖥  Ваш VPS     ║ ═══► 🌍 Интернет
║   (Android)      ║        155.212.*.*            ║     Серверы      ║                      ║     (Server)     ║
║                  ║                               ║                  ║                      ║                  ║
╚══════════════════╝                               ╚══════════════════╝                      ╚══════════════════╝
        Вы                                          Инфраструктура VK                          Ваш сервер
```

### Что видит оператор / ТСПУ

```diff
- Без WDTT:   Клиент ──── [VPN трафик] ──────────── VPS             ← ❌ блокируется / режется
+ С WDTT:     Клиент ──── [DTLS "звонок"] ────── 155.212.*.*        ← ✅ выглядит как звонок в VK
```

### Стек протоколов (послойно)

```
┌─────────────────────────────────────────────────┐
│              Ваш трафик (HTTP, DNS, …)          │  ← Данные приложений
├─────────────────────────────────────────────────┤
│              WireGuard (шифрование)             │  ← E2E шифрование до VPS
├─────────────────────────────────────────────────┤
│              DTLS 1.2 (шифрование)              │  ← Неотличимо от VK Calls
├─────────────────────────────────────────────────┤
│              TURN Channel Data                  │  ← Стандартный TURN relay
├─────────────────────────────────────────────────┤
│              UDP → 155.212.*.*:443              │  ← Серверы VK (медиа-трафик)
└─────────────────────────────────────────────────┘
```

## 🔄 Как это работает

```
┌──────┐         ┌──────────┐         ┌──────────┐         ┌─────────┐         ┌──────────┐
│  1   │         │    2     │         │    3     │         │    4    │         │    5     │
│ VK   │ ──────► │  TURN    │ ──────► │  DTLS    │ ──────► │  WG    │ ──────► │ Трафик  │
│ API  │         │  Alloc   │         │  Handshk │         │ Tunnel │         │  Идёт!  │
└──────┘         └──────────┘         └──────────┘         └─────────┘         └──────────┘
Получаем          Выделяем             Устанавл.           Поднимаем            Интернет
временные         relay на             шифрованный         WireGuard            работает
TURN-креды        TURN-сервере         канал               внутри DTLS          через VPS
```

| Шаг | Описание |
|:---:|----------|
| **1** | Клиент получает временные TURN-креденшлы через **VK Calls API** |
| **2** | На TURN-сервере VK выделяется relay-адрес для UDP-трафика |
| **3** | Устанавливается **DTLS-соединение** (шифрованный UDP) — для оператора это «звонок» |
| **4** | Внутри DTLS поднимается **WireGuard-туннель** до вашего VPS |
| **5** | Весь трафик: `Приложение → WG → DTLS → VK TURN → VPS → Интернет` |

<br>

## ⚡ Почему это быстро

| Фактор | Описание |
|--------|----------|
| 🚀 **Приоритизация оператором** | Медиа-трафик (звонки, видео) получает приоритет — больше полосы, меньше дропов |
| 📦 **UDP, не TCP** | DTLS работает поверх UDP — нет overhead от TCP-рукопожатий и ретрансмиссий |
| 🔗 **Параллельные соединения** | Множество TURN-каналов объединяются в один высокоскоростной туннель |
| ⚙️ **WireGuard** | Самый быстрый VPN-протокол — минимальный overhead, криптография в ядре |

## ✨ Возможности

<table>
<tr>
<td width="50%">

### 🔧 Ядро
- 🔗 **Многопоточность** — до 80 параллельных TURN-соединений
- 🔄 **Авторотация кредов** — смена TURN-кредов без разрыва
- 🐕 **Watchdog** — автоперезапуск при сбоях
- 🔒 **Full Cone NAT** — автонастройка NAT на сервере

</td>
<td width="50%">

### 📱 Приложение
- ✂️ **Split Tunneling** — исключение приложений из VPN
- 🚀 **Deploy** — установка сервера н�� VPS из приложения по SSH
- 🤖 **Telegram Bot** — управление паролями и мониторинг
- 🔐 **Привязка устройств** — один пароль = одно устройство

</td>
</tr>
</table>

> [!WARNING]
> **Рекомендуемое количество соединений: 8–32.** Использование более 32 параллельных TURN-соединений повышает риск обнаружения аномального поведения со стороны VK и может привести к временной блокировке аккаунта или TURN-кредов. Максимум (80) предусмотрен для экспериментов — используйте на свой страх и риск.

## 🚀 Быстрый старт

### Установка сервера (из приложения)

```
1.  📱  Откройте вкладку «Deploy»
2.  🖥  Введите IP, логин и пароль сервера
3.  🔑  Задайте пароль туннеля в «Секретах»
4.  ▶️  Нажмите «Установить»
```

> Скрипт автоматически установит WireGuard, настроит NAT, файрволл и создаст systemd-службу.

### Подключение

```
1.  🔑  Введите пароль туннеля
2.  ▶️  Нажмите «Подключить»
3.  ✅  Готово — трафик идёт через VPN
```
