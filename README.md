# WDTT — WireGuard over TURN Tunnel

Высококачественный VPN-туннель через TURN-серверы VK до собственного VPS.

Проект создан для **домашнего использования** — end-to-end туннель от Android-устройства до вашего сервера без сложных настроек. Способствует ускорению интернет-соединения за счёт приоритизации медиа-трафика операторами связи.

## Схема подключения

```
┌────────────┐        DTLS (UDP)        ┌──────────────┐       UDP        ┌────────────┐
│            │      (похож на звонок)    │              │    (WireGuard)   │            │
│   Клиент   │ ──────────────────────── │  VK TURN     │ ────────────── │  Ваш VPS   │ ─── Интернет
│  (Android) │    155.212.*.*           │  Серверы     │                 │  (Server)  │
│            │                          │              │                 │            │
└────────────┘                          └──────────────┘                 └────────────┘
     Вы                              Инфраструктура VK                   Ваш сервер
```

### Что видит оператор / ТСПУ

```
Без WDTT:    Клиент ──── [VPN трафик] ──── VPS          ← блокируется / режется
С WDTT:      Клиент ──── [DTLS звонок] ──── 155.212.*.* ← выглядит как звонок в VK
```

Оператор и ТСПУ видят **только DTLS-трафик** (протокол видеозвонков) до IP-адресов VK (`155.212.*.*`). Это неотличимо от обычного звонка в VK — тот же протокол, те же серверы, те же порты. Реальный VPN-трафик до вашего VPS **полностью инкапсулирован** внутри DTLS.

### Как это работает

1. **Клиент** получает временные TURN-креды через VK Calls API
2. Устанавливается **DTLS-соединение** (шифрованный UDP) через TURN-серверы VK
3. Внутри DTLS создаётся **WireGuard-туннель** до вашего VPS
4. Весь интернет-трафик идёт по цепочке: `Приложение → WireGuard → DTLS → VK TURN → VPS → Интернет`

### Почему это быстро

- Операторы **приоритизируют** медиа-трафик (звонки, видео) — ему выделяется больше полосы
- DTLS работает поверх **UDP** — нет overhead от TCP-рукопожатий
- До **80 параллельных** TURN-соединений объединяются в один туннель
- WireGuard — самый быстрый VPN-протокол с минимальными задержками

## Возможности

- **Многопоточность** — до 80 параллельных TURN-соединений для максимальной скорости
- **Автоматическая ротация** — периодическая смена TURN-кредов без разрыва соединения
- **Watchdog** — автоматический перезапуск при сбоях
- **Split Tunneling** — исключение приложений из VPN
- **Deploy** — установка сервера на VPS прямо из приложения через SSH
- **Full Cone NAT** — автоматическая настройка NAT на сервере
- **Telegram Bot** — управление паролями и мониторинг через бота
- **Привязка устройств** — один пароль = одно устройство

## Структура

| Компонент | Файл | Описание |
|-----------|------|----------|
| Go Client | `client.go` | TURN/DTLS клиент, диспетчер соединений |
| Go Server | `server.go` | WireGuard сервер, NAT, Telegram бот |
| Android App | `app/` | Jetpack Compose UI, управление туннелем |
| Deploy Script | `app/src/main/assets/deploy.sh` | Автоустановка сервера на VPS |

## Сборка

### Серверная часть (Go)

Требуется Go 1.21+:

```bash
go build -o server server.go
```

### Клиент (Android)

Требуется Android Studio с SDK 26+:

```bash
# Собрать Go-клиент как .so для Android
gomobile bind -target=android -androidapi 26 -o app/libs/client.aar

# Собрать APK
./gradlew assembleRelease
```

### Установка сервера

Сервер можно установить прямо из Android-приложения:

1. Откройте вкладку **Deploy**
2. Введите IP, логин и пароль сервера
3. Задайте пароль туннеля в **Секретах**
4. Нажмите **Установить**

Скрипт автоматически установит WireGuard, настроит NAT и создаст systemd-службу.
